# 要件定義: 記事品質改善 v2（全4施策）

## ユーザーストーリー

AS A ボートレース予想に興味がある読者
I WANT TO AIの予測根拠と実績を理解したい
SO THAT 信頼できるAI予想サービスか判断し、有料記事を購入したい

## 施策一覧

### 施策1: 的中レースの「なぜ」深掘り
- 3連単的中レースに対し、予測時のデータ（モーター勝率、コース別成績、平均ST等）から的中理由を自動生成
- accuracy_log + predictions テーブルからデータ取得
- 1記事あたり上位2〜3本の的中レースを深掘り

### 施策2: 累計実績グラフ画像
- 過去30日の的中率・ROI推移をグラフ化（Playwright HTML→PNG方式）
- 結果記事・実績記事のアイキャッチ or 本文内に埋め込み
- matplotlib不要（既存のPlaywright HTML→PNG パイプラインを活用）

### 施策3: 結果記事の文字数増加
- 場ごとの傾向分析（インコース勝率、決まり手分布等）
- 当日の全体統計（レース数、1着的中数、ROI）のより詳細な表示
- 目標: 1,000〜2,000字 → 2,500〜3,500字

### 施策4: タイトルバリエーション
- 的中数・ROI・払戻額に応じてタイトルを動的変更
- 例: 3連単10本的中 → 「3連単10本的中！」を前面に
- 例: 高額払戻 → 「万舟的中！」を含む

## 受入条件

### 機能要件
- [ ] 的中レースに予測根拠テキスト（2〜4文）が付与される
- [ ] 的中根拠は predictions テーブルの analysis + 出走表データから生成
- [ ] 30日推移グラフがHTML→PNG→note.comアップロードで本文に表示される
- [ ] グラフは的中率(折れ線) + ROI(棒グラフ)の2軸表示
- [ ] 結果記事に「場ごとの傾向」セクションが追加される
- [ ] 決まり手データ（results.race_technique_number）を活用
- [ ] タイトルが当日の結果に応じて3パターン以上から選択される
- [ ] 既存テスト93件が全てパスする
- [ ] 新規テストが各施策に対して追加される

### 非機能要件
- パフォーマンス: グラフ生成はPlaywright起動を含め30秒以内
- 互換性: note.com ProseMirrorエディタ対応（h2/h3/p/strong/ul/li/hr のみ）
- CI環境: GitHub Actions headless Chromiumで動作

## 制約事項
- フレームワーク: Python 3.11 + 既存パッケージ（新規依存追加なし）
- データ: SQLite DB内の既存テーブルのみ使用
- 画像生成: Playwright HTML→PNG（matplotlib等は追加しない）
- note.com HTML制約: table/blockquote/code タグ不可
