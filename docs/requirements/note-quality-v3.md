# 要件定義: note記事品質 価値最大化 v3

## 競合分析サマリー

### 市場構造
| セグメント | 代表 | フォロワー | 価格帯 | 特徴 |
|-----------|------|-----------|--------|------|
| プロ認定AI | yuu_noteid | 2,318 | 3,000-5,000円/月 | 40万レース学習、netkeiba認定 |
| ナイター特化 | lemon_house | 693 | 250円/本 + 5,000円/月 | 1,630記事、レース単位 |
| 高額パーソナル | mana_kyoutei | - | 25,000-100,000円 | 感情訴求、ストーリー重視 |
| チーム型 | bright_iguana224 | - | 1,600円/本 | 52-89いいね、マガジン活用 |
| 水理AI（自社） | suiri_ai | 0 | 300円/本 + 1,000円/月 | 全自動AI、透明性高い |

### 自社の強み（活かせていない）
1. **完全自動365日運用** — 競合は手動投稿が主流
2. **全データ公開** — 損失も隠さない透明性（競合は勝ちだけ強調）
3. **技術的裏付け** — LightGBM + 特徴量エンジニアリング（競合はAI内容不明）
4. **全場全レース網羅** — 一部場のみの競合が多い

### 自社の弱点（致命的）
1. **フォロワー0** — コンテンツ品質以前に発見されていない
2. **記事が「報告書」** — 感情・ストーリーがなく、読み物として楽しくない
3. **差別化が伝わらない** — 「365日自動」「全履歴公開」が埋もれている
4. **CTA弱い** — フォロー・購入への導線が弱い
5. **視覚的に貧弱** — 競合のemoji/装飾に対してプレーンすぎる

## ユーザーストーリー

AS A 競艇ファン（note.com読者）
I WANT TO 信頼できるAI予測の結果を分かりやすく読みたい
SO THAT 明日の予測にも期待してフォロー・購読したくなる

## 施策一覧（優先順）

### 施策A: 記事の「読み物化」— ストーリーテリング強化
**目的**: 報告書→読み物への転換。感情を動かすコンテンツに
- [ ] A1: 結果記事の冒頭に「今日の一言」導入（勝ち: 盛り上げ / 負け: 正直に反省）
- [ ] A2: 3連単的中レースに「ドラマ」を付与（展開描写: 「3号艇がまくり差しで鮮やかに抜け出した」等）
- [ ] A3: 負けパターンの分析追加（「今日の敗因」セクション — 荒れた場・逆転展開の傾向）
- [ ] A4: 翌日プレビュー強化（「明日の注目場」に具体的理由を付与）

### 施策B: 視覚的インパクト強化
**目的**: スクロールを止める装飾。note.com制約内で最大限の視覚効果
- [ ] B1: セクション区切りにUnicode装飾（━━━ ◆ ━━━ 等）
- [ ] B2: 数値ハイライトに記号活用（◎◯△✕ + 絵文字控えめ使用）
- [ ] B3: 擬似テーブルを全データセクションに適用（`<ul><li>` + 全角揃え）
- [ ] B4: ランク別の視覚的マーカー（S:◎ A:○ B:△ C:✕）

### 施策C: エンゲージメント導線強化
**目的**: フォロー→購読への明確な導線設計
- [ ] C1: 記事冒頭に「フォローすると毎朝7:30に予測が届く」を太字で配置
- [ ] C2: 結果記事の末尾に「明日のSランク予測は朝7:30公開」カウントダウン的CTA
- [ ] C3: 関連記事リンクをカード風に装飾（現在はプレーンURL）
- [ ] C4: メンバーシップ誘導に実績数値を直接引用（「先月の3連単〇本的中を全部読めます」）

### 施策D: SEO・発見性向上
**目的**: note.com内検索とGoogle検索での発見性向上
- [ ] D1: タイトルに感情フック追加（現在: データのみ → 「万舟的中！」等は実装済み、さらに強化）
- [ ] D2: 記事末尾に次回予告（「明日は〇〇場のSGレースに注目」等）
- [ ] D3: ハッシュタグの最適化（競合で効果の高いタグ分析）

### 施策E: 信頼性の可視化
**目的**: 「このAIは本物」を読者に即座に伝える
- [ ] E1: 「水理AIとは」セクションに具体数値追加（累計〇万レース分析、運用〇日目）
- [ ] E2: 累計実績に「連続運用日数」バッジ追加
- [ ] E3: 負けも含めた全履歴公開をアピール（「勝ちだけでなく負けも全公開」）

## 受入条件

### 機能要件
- [ ] 全5記事タイプ（grades/results/midday/track_record/membership）に施策適用
- [ ] note.com ProseMirror制約を遵守（h2/h3/p/strong/ul/li/hr のみ）
- [ ] 既存テスト（365件）が全パス
- [ ] 新機能に対するテスト追加

### 非機能要件
- パフォーマンス: 記事生成時間の増加は1秒以内
- 互換性: 既存のCLI/GitHub Actionsフローに変更なし
- 保守性: 記事テンプレートの文言はコード内定数で管理

## 制約事項
- フレームワーク: Python + note.com ProseMirror互換HTML
- データ永続化: SQLite（既存DB）
- 画像: note.com画像APIが404のため、テキストベースのみ
- 表: `<table>` 非対応 → `<ul><li>` 擬似テーブルで代替

## 実装スコープ（Phase 1 — 今回）
コスト対効果が最も高い施策に絞る:
1. **A1 + A3**: 冒頭の一言 + 敗因分析（読み物化の核）
2. **B1 + B3**: 装飾 + 擬似テーブル（視覚改善の核）
3. **C1 + C2**: フォロー/CTA強化（導線の核）
4. **E1**: 水理AIの信頼性数値化
